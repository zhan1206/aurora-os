# AuroraOS 系统架构设计文档

## 1. 概述

AuroraOS 是一个基于 x86_64 架构的自主研发操作系统内核，采用宏内核（Monolithic Kernel）设计，支持多任务调度、虚拟内存管理、虚拟文件系统、POSIX 信号机制和 ELF 可执行文件加载。

- **版本**: 2.5.0
- **目标架构**: x86_64 (AMD64)
- **内核类型**: 宏内核
- **启动方式**: Multiboot1 (GRUB2)
- **许可证**: MIT

## 4. 技术选型

| 技术领域 | 选型 | 理由 |
|----------|------|------|
| 启动协议 | Multiboot1 | GRUB2 广泛支持，简单可靠 |
| 内存模型 | 4 级分页 (PML4) | x86_64 标准，支持 48 位虚拟地址 |
| 物理内存分配 | Buddy 伙伴系统 | 高效的连续页分配与合并 |
| 内核堆分配 | Slab 分配器 | 小对象高效分配，减少碎片 |
| 进程调度 | 轮转调度 (Round Robin) | 简单公平，适合初期开发 |
| 文件系统 | VFS + RamFS | 抽象层支持多种文件系统，RamFS 适合无盘环境 |
| 系统调用 | SYSCALL 指令 (MSR) | x86_64 快速系统调用标准 |
| 编译工具链 | x86_64-elf-gcc | 交叉编译，无宿主依赖 |
| 链接脚本 | 自定义 linker.ld | 精确控制内核内存布局 |
| 构建系统 | GNU Make | 简单通用，适合内核项目 |

## 5. 内存布局

```
物理内存布局:
  0x00000000 - 0x00000FFF : IVT / BDA (保留)
  0x00001000 - 0x0009FFFF : 常规内存 (可用)
  0x000A0000 - 0x000FFFFF : VGA / BIOS (保留)
  0x00100000 - 0x3FFFFFFF : 内核 + 可用内存 (~1GB)

虚拟地址空间:
  0x00000000 - 0x00007FFFFFFFFFFF : 用户空间 (128TB)
  0xFFFF800000000000 - 0xFFFFFFFFFFFFFFFF : 内核空间 (128TB)
  0xFFFFFFFF80000000 : 内核代码段 (高半核)
```

## 6. 启动流程

```
1. GRUB2 加载 kernel.elf
2. entry.S: 设置栈指针，跳转到 kmain
3. kmain():
   a. console_init()     - 初始化 VGA 控制台
   b. gdt_init()         - 设置 GDT
   c. idt_init()         - 设置 IDT
   d. phys_mem_init()    - 初始化物理内存管理
   e. slab_init()        - 初始化 Slab 分配器
   f. page_table_init()  - 初始化页表
   g. syscall_init()     - 配置 MSR 系统调用
   h. sched_init()       - 初始化调度器
   i. keyboard_init()    - 初始化键盘驱动
   j. pit_init()         - 初始化定时器
   k. vfs_init()         - 初始化 VFS
   l. mount_ramfs()      - 挂载 RamFS
   m. shell_main()       - 启动 Shell
```

## 7. 关键设计决策

### 7.1 Copy-On-Write (COW)
- fork() 时父子进程共享物理页，标记为只读
- 写入时触发缺页异常，复制页面
- ref_count 跟踪共享计数，为 0 时释放物理页

### 7.2 内核抢占
- 当前采用协作式调度（yield）
- PIT 定时器中断触发调度检查
- 未来可扩展为抢占式

### 7.3 安全架构
- 用户/内核地址空间隔离（高半核）
- 页表保护：用户模式不可访问内核页
- NX 位防止用户态代码执行
- 能力系统提供细粒度访问控制
- ELF 入口点验证防止任意地址执行
- ELF 段边界验证防止越界内存访问
- VFS 路径遍历防护（拒绝 "." 和 ".." 组件）
- phys_to_virt 身份映射范围验证（0-1GB）
